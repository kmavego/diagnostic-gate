# Versioning

Проект использует SemVer: MAJOR.MINOR.PATCH.

## Gate Specs
- MAJOR: ломаем схему артефактов / смысл правил / совместимость исполнения
- MINOR: добавляем новые правила/поля без поломки существующих потребителей
- PATCH: уточнения/исправления без изменения схемы (например, пороги, тексты, дополнительные паттерны)

## Registries
- gates_registry.yaml / artifact_registry.yaml:
  - MAJOR: изменение структуры файла или контрактов ссылок
  - MINOR: добавление новых Gate/артефактов
  - PATCH: исправления, расширение примеров, уточнение описаний

## Deprecation
См. DEPRECATION_POLICY.yaml.
